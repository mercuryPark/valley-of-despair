# PRODUCT.md — 컨셉·정체성

## 한 문장 정의

FE 개발자가 실무에서 마주치는 핵심 4 도메인을 선수→심화 학습 경로 트리로 깊이 학습하고, 인접 BE/DB 기초까지 FE 시점에서 함께 정리하는 학습 플랫폼.

## 정체성

| 항목 | 결정 |
| --- | --- |
| 카테고리 | **학습 플랫폼** (학습 게임 아님) |
| 사용자 모드 | **자유 탐색** (선형 강제 진행 아님) |
| 콘텐츠 깊이 | **깊게** (얕은 위키 아님) |
| 평가 시스템 | **없음** (MVP 단계). v2에서 검토 |
| 랭킹 | **없음**. v3에서 검토 |
| 수익화 | **없음**. 포트폴리오 목적 |

## 타겟

- **주 타겟**: FE 1~5년차
- **부 타겟**: 비전공자 출신 주니어, FE 전향 준비자
- **제외**: 코딩 입문자 (다른 플랫폼이 더 적합)

## 동기 (왜 만드는가)

1. 코딩테스트 중심 채용 문화의 안티테제. 알고리즘 암기보다 **실무 감각**이 중요하다는 신념.
2. 기존 학습 플랫폼(인프런·Udemy)은 일방향 VOD, roadmap.sh는 너무 얕음. **깊고 자유 탐색 가능한** 학습 자료가 한국어로 부재.
3. 본인(Hoyeon, FE 4년차)의 실무 경험을 콘텐츠화하면서 **포트폴리오로 활용**.

## MVP 범위

### 메인 4 도메인 (각 15~20 노드, 총 60~80)

1. **렌더링** (Rendering)
   - 브라우저 렌더링 파이프라인, CRP, Reflow/Repaint, 합성, React 렌더링과의 차이
2. **상태관리** (State Management)
   - 로컬·서버 상태 분리, Zustand·React Query 패턴, 상태 동기화 안티패턴
3. **성능** (Performance)
   - 번들 크기, 렌더링 성능, 네트워크 성능, 측정·관측 도구
4. **비동기/네트워크** (Async / Network)
   - 이벤트 루프, Promise·async/await, HTTP, REST·GraphQL, 캐싱

### 보조 4 영역 (각 5~10 노드, 총 20~40)

FE 시점에서 알면 좋은 수준으로만 한정. 깊이 욕심 금지.

1. HTTP·네트워크 기초 (FE 시점)
2. 인증·인가 기초
3. DB 기초 (FE가 API 호출할 때 알면 좋은 수준)
4. 서버 요청 라이프사이클

### 합계: 80~120 노드

## 콘텐츠 합성 원칙

| 소스 | 역할 | 비중 |
| --- | --- | --- |
| 외부 자료 (MDN, web.dev, 공식 문서, 토스/카카오 기술블로그) | 1차 출처·정확성 보증 | 30% |
| LLM | 구조 초안·예시 코드·표현 다듬기 | 30% |
| 본인 실무 경험 | "실무에서 왜 중요한가"의 차별화 | **40%** |

**핵심**: 본인 경험 부분이 차별화의 원천. 빠지면 어디서나 볼 수 있는 글이 됨.

## 차별화

```
              roadmap.sh    인프런/Udemy    valleyofdespair
시각화        매우 강함      중간            중간 (사이드바+그래프)
콘텐츠 깊이   얕음           깊음 (강의)     깊음 (텍스트)
한국어        약함           강함            강함
실무 경험     없음           강사별          본인 경험 명시
자유 탐색     강함           약함            강함
```

차별화의 90%는 **콘텐츠 깊이 + 본인 실무 경험**에서 나온다. 시각화로 승부 안 봄.

## 명시적으로 거부한 방향

- ❌ FE 외 영역 깊이 다루기 (BE·DevOps·DB) — 본인 비전문, 검수 불가
- ❌ 평가·점수·랭킹 시스템 — "모호한 답을 점수화"의 신뢰성 문제, v2 이후
- ❌ AI 대화형 시뮬레이션 — LLM 페르소나 붕괴·비용·복잡도, v3 이후
- ❌ 학습 게임 UI (Three.js 트리, 5스텝 클리어 게이팅) — 자유 탐색 파괴
- ❌ "본인도 학습하면서 만들기" 동기로 비전문 영역 확장 — 학습은 별도, 만드는 건 잘 아는 것
- ❌ 웹 + 앱 동시 개발 — 1인 개발 부담, RN은 v2 이후

## 사용자 시나리오 (가상)

### 시나리오 1: "출퇴근길 학습"

지하철에서 휴대폰으로 PWA 접속 → 사이드바에서 "성능" 도메인 → "이미지 최적화" 노드 → 5분 분량 읽기 → 완료 체크.

### 시나리오 2: "면접 준비"

데스크톱에서 "렌더링" 도메인 그래프(v1.1) → CRP 노드부터 선수 관계 따라 학습 → 노드별 본문 + 본인 경험 부분 정독 → 다음 노드로.

### 시나리오 3: "트러블슈팅 중 빠른 참조"

검색창에 "reflow" → Pagefind가 관련 노드 즉시 표시 → 클릭해서 확인.

## v2~v3 로드맵 (참고용, MVP에 영향 없음)

- **v2**: 노드 단위 간단 퀴즈, 학습 통계, 객관식 평가 (CS 영역에 한해), BE/DevOps 도메인 확장 검토
- **v3**: AI 기반 동적 시나리오·대화형 시뮬레이션, 다축 평가, 종합 랭킹

이 항목들은 MVP에 일체 포함하지 않는다.
