# 0004. 학습 우선, 평가·AI는 v2~v3

- **Status**: Accepted
- **Date**: 2026-05-03

## Context

초기 컨셉은 "테스트·랭킹 + 다축 평가" 중심이었다. 검토 결과 다음 문제 발견:

1. **모호한 답을 점수화하는 신뢰성 문제**
   - "디자이너가 검색창 3px 옮겨달라고 하면?" 같은 문제는 정답이 모호
   - 점수 가중치는 결국 출제자 주관 → "출제자 취향 테스트"로 변질
   - 이전 시도(behavioral interview platform)들이 학습 도구로는 살아남되 평가/랭킹으로는 거의 실패

2. **콘텐츠 생산 비용**
   - 백준의 자동 채점 + 무한 테스트 케이스 같은 자동화 불가능
   - 1인 개발자가 시나리오 100개 + 평가 가중치 설계는 비현실적

3. **AI 시뮬레이션의 추가 리스크**
   - LLM 페르소나 붕괴, 비용, 평가 일관성 부재
   - 작성자 LLM API 직접 호출 경험 전무 → MVP에 포함 시 출시 무산 위험

## Decision

- **MVP (v1)**: 학습 플랫폼만. 평가·랭킹·AI 일체 없음.
- **v2**: 학습 흡수 검증용 객관식 퀴즈 (정답 명백한 CS 영역에 한해)
- **v3**: AI 기반 동적 시나리오, 다축 평가, 종합 랭킹 (시장 검증 후)

## Consequences

### 긍정

- "정답 신뢰성" 문제 MVP에서 사라짐
- 콘텐츠 부담이 시나리오 → 학습 자료로 변경 (1인 개발 가능 범위)
- LLM 의존 0 → 비용·복잡도 0
- 면접 어필 명확: "MVP는 학습 → 평가는 신뢰 쌓인 후 도입"

### 부정

- "테스트·랭킹" 같은 게이미피케이션 요소 부재 → 동기부여 장치 약함
- → 진척도 시각화·학습 통계로 일부 보완 (v1.5)

## 거부된 방향

### 평가 우선 + 랭킹 메인
- 신뢰성 문제 해결 못 한 채 출시 시 "장난 점수"로 인식되어 사용자 이탈
- 1인 사이드 프로젝트 단계에서 평가 신뢰성 쌓기 비현실적

### AI 시뮬레이션 MVP 포함
- LLM API 경험 전무 + 페르소나·평가 일관성 튜닝 = 끝없는 작업
- "데모는 1주, 프로덕션은 6개월" 격언
- v3로 명확히 미룸

## 학습 흡수 검증 보강 (v1.5)

평가·랭킹은 거부했지만 학습 흡수 확인은 필요. v1.5에서 다음 검토:

- 노드 단위 객관식 퀴즈 3~5 문항 (정답 명백한 CS 영역만)
- 모호한 협업·트러블슈팅은 퀴즈 X (일관된 채점 불가)
- 점수보다 *학습 흡수 자가 진단* 용도

## 재평가 트리거

- 출시 후 6개월 + 사용자 100명 이상 + 학습 흡수 데이터 축적 → v2 평가 도입 검토
- 그 전에는 학습 콘텐츠 깊이에만 집중
