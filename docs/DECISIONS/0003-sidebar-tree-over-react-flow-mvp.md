# 0003. 트리 시각화: 사이드바 (MVP) → React Flow (v1.1)

- **Status**: Accepted
- **Date**: 2026-05-03

## Context

학습 트리(80~120 노드)를 어떻게 시각화할 것인가. 후보:

1. 사이드바 계층형 트리 (구현 3~5일)
2. React Flow (xyflow) 그래프 (구현 2~4주)
3. Three.js 게임 UI 스타일 트리 (구현 8~16주)
4. 정적 SVG 다이어그램

## Decision

- **MVP**: 사이드바 계층형 트리만
- **v1.1**: 도메인별 React Flow 그래프 추가
- **랜딩 페이지 (v1 마지막)**: react-three-fiber로 3D 비주얼 (학습 트리 아님)
- **거부**: Three.js 학습 트리, 정적 SVG, 학습 영역 내 3D

## Consequences

### 긍정

- MVP 출시 일정 준수 (시각화에 4주 빠지지 않음)
- 모바일 친화 (사이드바 → 햄버거 메뉴)
- 자유 탐색 보장 (학습 플랫폼 정체성과 정렬)
- SEO·접근성 자연스러움 (`<nav>`, `<ol>` 구조)
- v1.1에서 도메인별 그래프 추가 시 점진 도입 가능
- 콘텐츠 중심 차별화에 시간 집중

### 부정

- 첫인상 임팩트 약함 (사이드바 자체는 흔함)
- → 랜딩 3D로 보완

## 거부된 대안

### Three.js 학습 트리 (게임 UI 스타일)
- **자유 탐색 파괴**: 5스텝씩 보여주고 클리어해야 진행은 학습 게임 모델
- 학습 플랫폼 정체성과 본질적 충돌
- 모바일 UX 거의 불가능
- SEO 제로 (검색 유입 차단)
- 접근성 재앙
- 노드 추가마다 3D 씬 재배치 필요
- 1인 개발 8~16주 추가 → MVP 출시 미룸
- 동기 손실 위험 큼

### React Flow MVP 도입
- 80~120 노드 자동 레이아웃 (Dagre/ELK) 학습 곡선
- 노드 위치 결정·성능 튜닝 비용
- v1.1로 미루면 사용자 피드백 보고 도입 자체를 재평가 가능
- → 시간을 콘텐츠 깊이에 투자하는 게 차별화에 유리

### 정적 SVG 다이어그램
- 노드 추가/이동마다 SVG 수정 필요
- 콘텐츠 진화 잦은 학습 플랫폼에 부적합

## 시각화 영역 경계

```
[랜딩 / ]                         3D OK (react-three-fiber)
[학습 영역 /learn/* ]              3D 금지, 사이드바 + 그래프
[노드 페이지 /learn/[d]/[s] ]      인터랙티브 다이어그램 (주제별)
```

학습 영역에 3D가 한 번 침투하면 끝없이 "여기도 좀 더 화려하게"가 됨. 명시적 금지.

## 시각적 매력 보강 포인트

사이드바만으로는 첫인상 약함을 보완:

1. **랜딩 페이지 3D**: 협곡·나무 모티프, 1주 예산
2. **타이포그래피·여백·색감 일관성**: 학습 사이트 매력의 90%
3. **노드 페이지 내부 인터랙티브 다이어그램**: 주제별 (이벤트 루프·렌더링 파이프라인 등)
4. **v1.1 도메인 그래프**: React Flow로 도메인당 15~20 노드 시각화

## 재평가 트리거

- 출시 후 사용자 피드백에서 "그래프 시각화 필요"가 명확히 모이면 v1.1에서 React Flow 도입
- 그렇지 않으면 도입 자체를 재평가 (안 해도 될 가능성도 큼)
